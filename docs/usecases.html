Use cases

CloudWall is best-suited for situations and scenarios that demand offline mode, quick UI tuning and flexible distributed DB to store data. 

Applications authored and deployed using CloudWall can transform CloudWall framework in nearly any kind of distributed system – social network, space for collaboration, content storage, CMS, net of data-collecting endpoints, CRM and so on.

Prototyping

CloudWall is extremely useful for UI prototyping. Complicated web app parts with real connectivity can be implemented and tested in hours. 

Most of code you write at CloudWall, is usable outside CloudWall – since apps are just jQuery.my manifests, that can be used anywhere, at any site. In general – just copy-paste manifest.

jQuery.my is not the thing every web engineer knows about – but its learning curve is very short, manifests are well readable pure js objects, and there are several example apps installed during first system run. 

Collecting data

CloudWall runs simple apps fast enough to use on lean tablets to collect data at real-life processes’ end points. Surely they can be not only tablets – anything that has HTML5 compliant browser can be an end point.

Once opened, browser tab with CloudWall need no internet connection – so apps can store data in offline mode and system will sync them when go online. Each end point device can be synced with its own DB in corporate or private cloud, and then all data are aggregated in separate DB.

Typical business-related forms of any structure can be authored, tested and deployed in about an hour or two.  All docs that CloudWall stores to DB are json + attached files. So data collecting form can hold attached pics, sounds or whatever.

As CloudWall is HTML5 system, all apps inside has full access to modern hardware-related HTML5 APIs. Geolocation, 3-axis gyro, microphone and cam – all are accessible from CloudWall apps. These features can enrich data being collected and in several cases even modify the whole process.

Idea from thin air. There exist several js libs, than use pre-learned neural networks to recognize something in real time on video stream from cam. They can be incorporated in CloudWall app in one-two-three.

Private clouds

One who controls data, controls entire process. Every more or less known cloud has server farm that holds the copy of all users’ data. Some of them can sell you their solution for private use – for several hundreds of thousands of bucks.

In fact, now any small business or even person can have own cloud for $0. Using CloudWall you can create and manage your own private cloud with your own access and sync policy. The only thing you need is web-connected machine with CouchDB running. Hardware requirements are very thin.

Duet CloudWall + CouchDB can be used to build private clouds for document-based collaboration of nearly any kind. Evernote-like scenarios are available out of the box, for more complex just write javascript app. Right in CloudWall.

All docs are synced with each private cloud subscriber immediately, conflicts in doc versions (if they occur) are stored to be manually resolved – so no data can be lost or overwritten silently.

One-app services

You can fork CloudWall and build custom bundle with one app inside. App can utilize CloudWall noBackend and UI capabilities – and be highly task-specific.

Removing configuration apps from bundle disables user from changing settings in simple way – so environment for the app can be rather restrictive.

Custom bundles can be pre-linked with private CouchDB instances – to your end-user devices synced right from start.
